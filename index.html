1. What is Version Control?

Version Control System (VCS) is a system that records changes to files over time so you can recall a specific version later. Think of it like an unlimited "undo" button for your entire project — every save is a snapshot you can return to.

Why do we need it?

  • History: See every change ever made and who made it.
  • Collaboration: Multiple developers can work on the same project without overwriting each other.
  • Backup: Code is safely stored in multiple places.
  • Experiment safely: Try new features in branches without breaking main code.
  • Revert mistakes: Roll back to any previous working state in seconds.

Types of Version Control

  1. Local VCS — keeps versions on your own machine only (e.g., RCS).
  2. Centralized VCS (CVCS) — single central server holds all versions (e.g., SVN, Perforce).
  3. Distributed VCS (DVCS) — every developer has a full copy of the repo (e.g., Git, Mercurial). This is the modern standard.
Real-life analogy: Version control is like Google Docs' "version history" but designed for code, with much more power.

2. What is Git?

Git is a free, open-source distributed version control system created by Linus Torvalds in 2005 to manage Linux kernel development. It tracks changes in your code and lets teams collaborate seamlessly.

Key features of Git

  • ⚡ Fast — most operations are local.
  • 🌐 Distributed — every clone is a full backup.
  • 🌿 Strong branching — cheap, fast, easy to merge.
  • 🔒 Data integrity — every change is SHA-1 hashed.
  • 💻 Works offline — commit without internet.

Git is a tool — you install it on your computer

$ git --version
git version 2.43.0

3. What is GitHub?

GitHub is a cloud-based hosting service for Git repositories. It adds a beautiful web UI, collaboration tools, issue tracking, pull requests, CI/CD (GitHub Actions), free static hosting (GitHub Pages), and much more on top of Git.

Founded in 2008, acquired by Microsoft in 2018, GitHub now hosts over 100 million developers and 420+ million repositories.

What can you do on GitHub?

  • 📁 Store your project code online safely.
  • 🗂 Manage versions of your projects with full history.
  • 🤝 Collaborate with developers worldwide.
  • 🌍 Host static sites (HTML/CSS/JS) for free with GitHub Pages.
  • 🐛 Track issues and manage projects with Kanban boards.
  • ⚙️ Automate tests and deployments with GitHub Actions.
  • 👁 Showcase your portfolio to recruiters.

7. Installing & Configuring Git

Install

  • Windows: Download from git-scm.com and install Git Bash.
  • macOS: brew install git
  • Linux: sudo apt install git

First-time configuration

$ git config --global user.name "Your Name"
$ git config --global user.email "you@email.com"
$ git config --global init.defaultBranch main
$ git config --list   # verify

8. Essential Git Commands (with examples)

Open a terminal inside any project folder and run these:

🔹 git init — start a new repository

$ git init
Initialized empty Git repository in /your/project/.git/

Creates a hidden .git folder that tracks everything.

🔹 git status — see what's changed

$ git status
On branch main
Untracked files:
  index.html
  style.css

🔹 git add — stage files for commit

$ git add index.html         # add one file
$ git add .                  # add ALL changed files
$ git add *.js               # add by pattern

🔹 git commit — save a snapshot

$ git commit -m "Initial commit: add homepage"
$ git commit -am "Quick: stage tracked + commit"
Tip: Write commit messages in imperative mood — "Add login", not "Added login".

🔹 Connect to a GitHub repo

1. Create a new empty repo on github.com → copy the URL.

2. Link your local folder:

$ git remote add origin https://github.com/username/repo.git
$ git remote -v          # verify remote

🔹 git push — upload commits to GitHub

$ git branch -M main
$ git push -u origin main   # first push (sets upstream)
$ git push                  # next pushes

🔹 git clone — copy a remote repo to your PC

$ git clone https://github.com/username/repo.git
$ cd repo

🔹 git pull — fetch + merge latest changes

$ git pull origin main

🔹 git log — view commit history

$ git log
$ git log --oneline --graph --all

10. Host Your HTML on GitHub Pages (Free Public URL)

GitHub Pages turns any repository with an index.html into a live website — completely free.

  1. Push your HTML/CSS/JS files to a GitHub repository (make sure index.html is in the root).
  2. Go to repo → Settings → Pages (left sidebar).
  3. Under Source, choose branch main and folder / (root).
  4. Click Save. Wait ~30 seconds.
  5. Your site is live at: https://username.github.io/repo-name/
Pro tip: You can also add a custom domain (e.g., yourname.com) in the same Pages settings.

12. Branches, Merging & Pull Requests

Branches let you work on features safely without breaking the main code.

$ git switch -c feature-login    # create + switch
# …make changes, commit…
$ git push -u origin feature-login

On GitHub, click Compare & pull request → reviewers approve → Merge pull request → done!

Resolving merge conflicts

If two people edit the same line, Git shows:

<<<<<<< HEAD
your changes
=======
their changes
>>>>>>> feature

Edit the file to keep the right code, then:

$ git add .
$ git commit -m "resolve merge conflict"
